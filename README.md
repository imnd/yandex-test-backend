# Парсер отзывов Яндекс.Карт (Laravel 11 + Vue 3 SPA + PostgreSQL)

Проект состоит из двух раздельных частей:
1. **`backend`**: Laravel 11 API с базой данных PostgreSQL и парсером на Playwright.
2. **`frontend`**: Vite Vue 3 (Composition API) Single Page Application.

---

## Выбор технологии парсинга

Парсер реализован с использованием библиотеки **Playwright-Chromium** в Node.js, которая вызывается из Laravel-сервиса `YandexParserService`.

### Почему именно Playwright?
1. **Динамический контент (SPA):** Яндекс.Карты — это тяжелое SPA-приложение. Отзывы не приходят в исходном HTML, а подгружаются динамически с помощью JavaScript при прокрутке. cURL или Guzzle на PHP могут прочитать только пустой HTML-каркас.
2. **Обход защиты от ботов:** Яндекс использует продвинутые анти-бот скрипты (анализ canvas, User-Agent, cookie, отпечатки). Playwright с настроенными аргументами запуска, реальным User-Agent, отключением флага автоматизации (`navigator.webdriver`) и имитацией человеческого поведения (случайные задержки 0.8-1.5 сек между прокрутками) позволяет обходить проверки и собирать данные без блокировок.
3. **Безопасность API-запросов:** Попытки повторить внутренние запросы к API Яндекса (`/maps/api/business/fetchReviews`) крайне нестабильны. Эти запросы защищены динамическими CSRF-токенами и параметром `s_hash`, подпись для которого генерируется запутанным JS-кодом Яндекса. Имитировать генерацию подписи на чистом PHP невозможно без выполнения JS.

## Структура Базы Данных (PostgreSQL)

Для снижения нагрузки и предотвращения частых блокировок со стороны Яндекса, результаты парсинга кэшируются в БД.
- **`organizations`**: Хранит URL, уникальный `yandex_id` организации, имя, средний рейтинг, количество оценок, количество отзывов и лог статуса парсинга (`pending`, `processing`, `completed`, `failed`).
- **`reviews`**: Таблица отзывов, связанных с организацией (`onDelete('cascade')`). Хранит имя автора, аватар, оценку (1-5 звезд), дату (строкой Яндекса) и текст отзыва.

## Постраничная навигация (Пагинация)

Отзывы выводятся порциями по **50 штук на страницу**. 
- В соответствии с лучшими практиками разработки, вся выборка из БД пагинируется на бэкенде через стандартный метод Eloquent `paginate(50)`. 
- Это предотвращает передачу лишних данных по сети (~600 отзывов весят много) и обеспечивает быструю работу.
- На фронтенде переключение страниц происходит асинхронно через Axios без перезагрузки всей страницы, с отображением красивых skeletons-заглушек в процессе перехода.

## Работа с Прокси-Серверами

Для обхода возможных лимитов и блокировок Яндекса по IP-адресу, парсер поддерживает ротацию прокси-серверов (как HTTP, так и SOCKS5).

### Как это устроено:
1. **База данных прокси:** Все прокси-серверы хранятся в таблице `proxies` БД.
2. **Ротация:** При запуске задачи парсинга оркестратор выбирает случайный активный прокси-сервер из базы данных.
3. **Обработка ошибок:** Если конкретный прокси возвращает ошибку 3 раза подряд, он автоматически деактивируется (`is_active = false`), предотвращая повторные сбои. При успешном парсинге счетчик ошибок прокси сбрасывается в `0`.

### Автоматическое обновление прокси:
В бэкенд встроена консольная команда для автоматического получения и обновления прокси из нескольких надежных публичных внешних списков (ProxyScrape, репозитории GitHub):
```bash
php artisan proxies:fetch
```

- **Автоматический запуск при старте:** Данная команда прописана в `docker-entrypoint.sh` и запускается при старте контейнера сразу после применения миграций и сидеров, гарантируя наличие свежих рабочих прокси на старте.
- **Периодический запуск:** Задача зарегистрирована в планировщике Laravel в файле `routes/console.php` и автоматически запускается **каждый день (daily)**.
- **Умное обслуживание БД:** Команда автоматически удаляет нерабочие прокси (с 3+ ошибками) и ограничивает максимальный размер таблицы до **2000 записей** (удаляя самые старые по времени последнего использования), что исключает раздувание базы данных.

## Инструкция по запуску

### Быстрый запуск на Windows (в корне проекта)

После первой настройки базы данных и установки зависимостей (см. шаги ниже), вы можете использовать скрипты запуска:
- **`run-all.bat`** — запускает в отдельных окнах API сервер, обработчик очередей и фронтенд сервер.
- **`run-frontend.bat`** — запускает только веб-интерфейс Vue 3.
- **`run-backend.bat`** — запускает API бэкенд и очередь Laravel.

---

### Шаг 1: Настройка Базы Данных

Создайте базу данных в PostgreSQL:
```sql
CREATE DATABASE yandex_parser;
```

### Шаг 2: Запуск Бэкенда (Laravel)

1. Перейдите в папку `backend` и настройте `.env`:
   ```bash
   cd backend
   cp .env.example .env
   ```
   Убедитесь, что настроены параметры PostgreSQL и Sanctum:
   ```env
   DB_CONNECTION=pgsql
   DB_HOST=127.0.0.1
   DB_PORT=5432
   DB_DATABASE=yandex_parser
   DB_USERNAME=postgres
   DB_PASSWORD=postgres

   SESSION_DOMAIN=localhost
   SANCTUM_STATEFUL_DOMAINS=localhost:5173,127.0.0.1:5173
   FRONTEND_URL=http://localhost:5173
   ```

2. Установите PHP и Node зависимости, а также браузер Playwright:
   ```bash
   composer install
   php artisan migrate --seed
   
   # Установка зависимостей парсера
   cd parser
   npm install
   npx playwright install chromium
   cd ..
   ```
   *(При сидировании создается пользователь: **`admin@example.com`** / **`password`**)*

3. Запустите локальный сервер Laravel:
   ```bash
   php artisan serve
   ```
   *(Будет запущен на http://localhost:8000)*

4. Запустите очередь Laravel для фонового парсинга:
   ```bash
   php artisan queue:work
   ```

### Шаг 3: Запуск Фронтенда (Vue 3 SPA)

1. Перейдите в папку `frontend`:
   ```bash
   cd ../frontend
   npm install
   ```
2. Запустите dev-сервер:
   ```bash
   npm run dev
   ```
   *(Будет запущен на http://localhost:5173)*

3. Откройте `http://localhost:5173` в браузере, авторизуйтесь и введите ссылку на Яндекс.Карты.
